이런 식으로 데스크탑처럼 의도대로 하단에 Footer가 나와야 하는데 `100vh`가 모바일에서는 다른 식으로 동작하여 Footer 부분이 잘리고 있습니다.


# 🔧 해결

해당 이슈는 `100vh` 사용 시에 모바일에서는 검색창과 하단에 생성되는 네비게이션 바를 뷰포인트에 포함시켜 발생되는 이슈로 모바일 최적화용인 `dvh`를 사용하여 해결해봅시다.

> dvh(dynamic viewport height)
> 주소 표시줄에 표시 여부에 따라 가변적으로 변하는 현재 보여지는 뷰포트 높이를 동적으로 가져온다.
> 참고로 safari와 edge, chrome 최신 버전에는 모두 지원하지만, 구버전과 firefox, IE에는 동작하지 않음.


```ts
export const layoutStyle = style({
  width: "100%",
  height: "100dvh",
  overflow: "auto",
});
```
